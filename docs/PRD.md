# 또간집 - Product Requirements Document (PRD)

## 1. 한줄 요약

**"재방문이 많은 음식점 = 진짜 맛집"** — 재방문 횟수를 핵심 신뢰 지표로 삼아, 지도 위에서 검증된 맛집을 발견하는 서비스.

## 2. 문제 정의

- 블로그/SNS 맛집 리뷰는 광고·협찬이 많아 신뢰도가 낮음
- 별점은 조작 가능하지만, **재방문은 거짓말하지 않는다**
- 진짜 맛집을 알려면 "같은 사람이 여러 번 간 곳"을 봐야 함
- 하지만 재방문 데이터를 수집할 수단이 없었음

## 3. 핵심 아이디어

### 3.1 재방문 = 맛집의 증거
- 같은 사람이 같은 음식점을 N회 이상 방문 → 재방문 맛집으로 인정
- 재방문 횟수가 높을수록 리스트에서 상위 노출

### 3.2 영수증 인증 방문 시스템
- 허위 등록 방지를 위해 **영수증 사진 촬영**으로만 방문 인증 가능
- **브라우저 OCR** (Tesseract.js)로 가게명 자동 추출
- DB 식당명과 자동 매칭 → 매칭 성공 시 포인트 지급
- 같은 날 같은 가게 중복 인증 방지

### 3.3 포인트 경제 (음식점명 잠금 해제)
- **문제:** 사용자가 방문 인증을 귀찮아할 수 있음
- **해결:** 방문 인증할 때마다 포인트 지급
- **포인트 사용:** 모든 맛집의 **이름을 확인**하는 데 5P 차감 (출처 무관)
- **잠금 상태:** "🔒 잠긴 맛집" + 카테고리 + 지역(구 단위) + 재방문 횟수만 표시
- **해제 상태:** 정확한 상호명 + 주소 + 카카오맵 링크 + 업무추진비 내역(해당 시)

### 3.4 초기 시드 데이터
- **국회의원 업무추진비** (29개 식당, 344건 지출 내역)
- **서울시 공무원 업무추진비** (1,159개 식당, 크롤링 데이터)
- 총 **1,188개 재방문 식당** (전부 카카오맵 placeId 보유)
- 모든 시드 데이터도 잠금 상태 — 5P로 열람

## 4. 타겟 사용자

- **탐색자:** "이 동네 진짜 맛집이 어디야?" → 재방문 지도 탐색
- **인증자:** 자기가 아는 맛집을 인증하고 포인트 획득 → 다른 맛집 발견에 사용
- **호기심형:** "국회의원이 5번이나 간 고깃집이 어디지?" → 초기 유입

## 5. 핵심 기능 (구현 현황)

### 5.1 재방문 맛집 지도 ✅
- 카카오맵 위에 재방문 맛집 드롭핀(물방울형) 표시
- 핀 번호 = 리스트 인덱스 매핑
- 카테고리 필터 (한식/중식/일식/양식/카페/술집/기타)
- 소스 필터 (전체 / 🏛️ 국회의원 / 🏙️ 서울시 공무원)
- **잠금 상태:** 지도 핀 없음, 리스트에만 표시 (🔒 + 카테고리 + 지역 + 재방문 횟수)
- **해제 상태:** 지도 핀 + 상호명 + 주소 + 카카오맵 링크

### 5.2 검색 기능 ✅
- 디바운스 검색 (300ms, 2자 이상)
- DB 식당 + 카카오 Local API 통합 검색
- 뷰포트 범위 제한 (`rect` 파라미터)
- 검색 결과가 리스트 영역 대체
- 검색 결과에서도 잠금/해제 UX 동일

### 5.3 영수증 인증 ✅
- 카메라/갤러리에서 영수증 업로드
- 브라우저 OCR (Tesseract.js) — 프로그레스바 표시
- 서버에서 DB 식당명 매칭 (정방향 + 역방향)
- 첫 방문 +10P, 재방문 +20P
- 같은 날 중복 방지
- 헤더 📸 버튼으로 진입

### 5.4 포인트 시스템 ✅
| 활동 | 포인트 |
|------|--------|
| 첫 방문 인증 | +10P |
| 재방문 인증 | +20P |
| 음식점명 열람 | -5P |
| 데모 계정 초기 | 1000P |

### 5.5 상세 모달 ✅
- 식당명, 주소, 카테고리, 출처, 재방문 횟수
- 카카오맵 링크 (placeId 있으면 직접 링크, 없으면 검색 링크)
- 국회의원 식당: 업무추진비 내역 (최신순 정렬)

### 5.6 마이페이지 ✅
- 프로필, 보유 포인트
- 잠금해제 식당 목록
- 포인트 사용 내역
- 다크모드 토글
- 로그아웃

### 5.7 사용자 인증 (부분 구현)
- **현재:** 데모 로그인 (쿠키 기반, demo@ttoganjip.kr)
- **예정:** OAuth (Google/Kakao) — NextAuth.js

## 8. 핵심 사용자 여정

### 시나리오 1: 탐색자
1. 사이트 접속 → 지도에 재방문 맛집 핀들 보임
2. "🔒 잠긴 맛집 · 한식 · 중구 · 재방문 5회" 카드 클릭
3. 잠금해제 모달 → 5P 차감 → 상세 모달 자동 오픈
4. 카카오맵 링크로 위치 확인 → 방문

### 시나리오 2: 인증자
1. 식당 방문 후 헤더 📸 클릭 → 영수증 촬영
2. 브라우저 OCR → 식당 매칭 → +10P (첫방문) / +20P (재방문)
3. 포인트로 다른 맛집 열람

### 시나리오 3: 호기심형
1. "국회의원이 7번 간 고깃집" → 소스 탭에서 🏛️ 국회의원 선택
2. 잠금해제 → 업무추진비 내역까지 확인

## 9. 성공 지표

- 영수증 인증 건수 / DAU
- 재방문 인증 비율 (전체 인증 중 재방문 비율)
- 포인트 소비율 (발급 대비 사용)
- 잠금해제 전환율
- 검색 이용률
